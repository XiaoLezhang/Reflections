查看文件的两个版本之间的差异如何帮助你了解引入的错误？
1，能够通过查看两个版本之间的差异缩小找出错误的范围
2.通过缩小范围以节省查找时间。
3.能提高找出引入的错误的效率

从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？
1.便于线性比较文件的不同版本之间做出了哪些更改。
2，当对更改的文件不满意时，用户可以轻松还原到某一版本的文件。
3，可以方便地对原文件编辑为多个不同的版本。

你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
手动提交的优点是可以自己控制提交的时间点，方便在自己觉得合适的时候提交。缺点是会有自己忘记提交的风险。
自动提交能够保证定时提交从而不会遗漏提交版本，缺点是时间是固定的，不能自主调节。

何你认为一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
因为有些文件会关联有其他文件。而像文档编辑类的多数都是单线程操作的。

如何使用命令 git log 和 git diff 来查看文件的历史记录？
请注意，运行 git log 会列出最近的提交及其相关信息（包括提交 ID）。运行 git diff（后跟两个提交 ID）会比较这两个提交的代码版本。如果需要复习，可以重新观看此视频.

使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
用了版本控制，当你因做出重大更改而出现bug时，可以检出代码的旧版本，这样就不用担心因做出错误的更改而无法还原到以前的版本了。
